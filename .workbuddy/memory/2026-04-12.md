# 2026-04-12 工作日志

## 15:30 Asia/Shanghai | 步骤：M8 三项 Bug 修复

- **摘要**：修复 build 后三个运行时问题：DevTools 弹窗、白屏不渲染、文件选择窗口失效
- **根因分析**：
  1. DevTools 弹窗：`mainWindow.webContents.openDevTools()` 未做 `app.isPackaged` 守卫，导致 dev 环境下调试窗口出现（实际上只在 VITE_DEV_SERVER_URL 时触发，已加守卫保险）
  2. 白屏：`sandbox: true` 会严格限制 preload 脚本能力，preload 中的 `ipcRenderer` 在 sandbox 下行为异常；同时渲染器路径未做 `app.isPackaged` 区分，改用 `app.getAppPath()` 统一处理
  3. 文件选择窗口：`registerIpcHandlers(mainWindow)` 传入的是启动时的 BrowserWindow 实例，若窗口重建（activated）后旧引用可能失效；改为传入 getter `() => mainWindow`，IPC handler 每次调用时动态取最新窗口
- **涉及文件**：
  - `src/main/index.ts`：移除无守卫的 openDevTools；改用 `app.isPackaged` + `app.getAppPath()` 处理路径；传 getter 给 registerIpcHandlers；`sandbox: false`
  - `src/main/ipc/register-ipc.ts`：函数签名改为 `(getMainWindow: () => BrowserWindow | null)`，所有需要 win 引用的 handler 改为动态取
- **验证**：`npm run typecheck` 通过；`npm run build` 通过；`npm run dist` 成功产出 `release/MilkySea Setup 0.1.0.exe`（136MB）
- **未解决**：待用户安装验证三项修复是否彻底生效
